Overview

This setup demonstrates a simple but production-aligned CI/CD deployment pattern: code changes merged into master trigger a GitHub Actions workflow that publishes website assets to Azure Storage Static Website.

Azure Storage Static Website GitHub Actions CI/CD Azure CLI Branching + PR Merge
Open Live Endpoint

Architecture

GitHub Push / PR Merge (master)
→ GitHub Actions Runner (ubuntu-latest)
→ Upload artifacts to Azure Storage ($web)
→ Azure Static Website Endpoint serves content

Azure Static Website hosting serves files directly from the reserved $web container.

Deployment Implementation

The workflow checks out the repository and deploys the site using Azure CLI batch upload:

az storage blob upload-batch
--destination '$web'
--source website
--overwrite

Using --overwrite ensures repeatable deployments and consistent state.

Security & Reliability Notes

• Credentials are stored in GitHub Secrets (not hardcoded in the repo).
• Each deployment run is traceable through Git commit history + Actions logs.
• Changes follow a PR-based workflow to keep the release path controlled.
